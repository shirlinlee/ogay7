一些提醒：

  1. css-bootstrap，有任何css更新，請做在index.scss
  2. vue，很單純的vue，請把選單跟內容分開宣告，因為選單日後會被拆到php的include去使用
    意即頁面上會有
    var menu = new Vue({el:"#menu"})
    var contect = new Vue({el:"#contect"})
    這樣拆開的東西
  3. index.html、product.html，是現存版本的php組裝回來變成html了，對你來說參考價值應該只剩一些元件跟meta裡面的東西，不用太拘泥，有一些加入最愛等function到時切版玩回來都會在重串，所以只要考慮切版問題就好
  4. pc跟m版的切換，請使用bootstrap以下的class進行控制

JSON檔案

  1. 首頁用index.json，照manson順序編排下來，有6個區塊，四種樣式
  2. 請把這四種樣式請做成component，分別為：head-slider、slide、tab-style-1、tab-style-2，日後能透過index.json內的type去判斷要出現哪種樣式
  3. block_id、tab_id、row_id不用理會，是後台用的識別
  4. product.json是商品頁用，這份檔案資料進來後，請encode urlcode再用
  5. 選單json為
    1. 主選單：find_main_list.json
    2. 子選單：find_sub_list.json
    3. 其他選單：find_extra_list.json
    目前的邏輯是，頁面loading進來>>呼叫主選單、點擊主選單>>呼叫子選單、點擊子選單>>呼叫其他選單，都是點擊以後，才會得知下一層選單，可以從正式機的這個網址
    https://www.7yago.com.tw/category?category_id=main-3c ，去看看呼叫的方式
    附件中的json檔為
    子選單：族群保健(亦即使用者在主選單點擊了族群保健，子選單出現族群保健底下項目)
    其他選單：女仕保健(亦即使用者在其他選單點擊了女仕保健，其他選單出現女仕保健底下項目)
